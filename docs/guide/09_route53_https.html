1 - AWS Route 53 호스팅 영역 생성 가이드

1. Route 53 콘솔 접속

AWS Route 53 콘솔에서 좌측 메뉴에서 "호스팅 영역"을 클릭합니다.

2. 호스팅 영역 생성 클릭

"호스팅 영역 생성(Create hosted zone)" 버튼을 클릭합니다.

3. 도메인 이름 입력

  • 도메인 이름(Domain name): 등록한 도메인을 입력 (예: example.com)
  • 유형(Type): 공개 호스팅 영역(Public hosted zone) 선택

4. 생성 완료 및 NS, SOA 레코드 확인

생성이 완료되면 기본적으로 다음 레코드가 자동 생성됩니다:

  • NS (Name Server) 레코드 → 도메인 등록기관에 설정할 네임서버 목록
  • SOA (Start of Authority) 레코드

5. 도메인 등록기관에 네임서버 설정

Route 53에서 생성된 NS 레코드의 값/트래픽 라우팅 대상의 4가지 항목 모두를 도메인을 등록한 사이트 가비아에 접속하여 네임서버로 등록합니다.

2 - AWS Certificate Manager 인증서 생성 방법 (HTTPS 인증서)

1. AWS Certificate Manager 콘솔 접속

자신이 원하는 리전에 인증서 요청을 클릭합니다.

2. 인증서 유형

퍼블릭 인증서 요청을 그대로 선택 한 채, 다음을 누릅니다

3. 도메인 이름

완전히 정규화된 도메인 이름

*.doitdoit.store 처럼 자신이 Route 53에 등록한 도메인 앞에 *을 반드시 붙여 줍니다.

나머지는 모두 그대로 하고 요청을 누릅니다.

4. 도메인

Route 53에서 레코드 생성을 누릅니다.

하나의 항목이 보인다면 그대로 레코드 생성합니다.

안보인다면, 해당 도메인에 이미 인증서가 있을 확률이 높습니다.
